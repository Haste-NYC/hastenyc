### Iteration: US-007 - Create API endpoint for Stripe checkout session
**Date**: 2026-01-20
**Status**: Complete

**What was implemented**:
- Created api/create-checkout-session.ts with serverless function for Stripe checkout session
- Function accepts POST with priceId and customerEmail in body
- Creates Stripe checkout session with mode: subscription
- Sets success_url to /checkout/success?session_id={CHECKOUT_SESSION_ID}
- Sets cancel_url to /checkout
- Returns session URL for redirect
- Created server.js with Express for local API development
- Added express, cors, dotenv packages
- Added npm run server script to package.json
- Updated .env.example with API_PORT and FRONTEND_URL placeholders

**Learnings/Patterns discovered**:
- Express 5 uses native ES modules (import/export) since package.json has "type": "module"
- dotenv.config({ path: '.env.local' }) for loading local environment files
- Stripe apiVersion parameter now uses format '2025-05-28.basil'
- API server runs on port 3001 while Vite dev server runs on 5173

**Files changed**:
- api/create-checkout-session.ts (new)
- server.js (new)
- package.json (updated - added server script, express/cors/dotenv deps)
- package-lock.json (updated)
- .env.example (updated - added API_PORT, FRONTEND_URL)
- .env.local (updated - added API_PORT, FRONTEND_URL)

### Iteration: US-008 - Integrate Stripe checkout redirect
**Date**: 2026-01-20
**Status**: Complete

**What was implemented**:
- Created src/lib/stripe.ts with loadStripe singleton initialization using publishable key from config
- Created src/hooks/useCheckout.ts hook that handles checkout API call with loading/error states
- Hook calls /api/create-checkout-session endpoint with priceId and customerEmail
- On successful response, redirects to Stripe checkout URL via window.location.href
- On error, displays toast notification using Sonner
- Updated Checkout.tsx to use useCheckout hook
- Button shows Loader2 spinner and "Redirecting to Stripe..." text during loading
- Added VITE_API_URL environment variable for configuring API endpoint

**Learnings/Patterns discovered**:
- Use singleton pattern for loadStripe to avoid creating multiple Stripe instances
- VITE_API_URL allows configuring API endpoint (defaults to localhost:3001)
- Sonner toast.error() for user-friendly error notifications
- Loader2 from lucide-react with animate-spin for loading spinners

**Files changed**:
- src/lib/stripe.ts (new)
- src/hooks/useCheckout.ts (new)
- src/pages/Checkout.tsx (updated - added useCheckout hook integration with loading state)
- .env.example (updated - added VITE_API_URL)
- .env.local (updated - added VITE_API_URL)
