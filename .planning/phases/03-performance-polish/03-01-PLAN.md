<task type="auto">
  <name>Task 1: Install optimization dependencies and configure image optimizer</name>
  <files>package.json, vite.config.ts</files>
  <action>
Install required dev dependencies:
```bash
npm install -D vite-plugin-image-optimizer sharp
```

Update vite.config.ts to add ViteImageOptimizer plugin:
- Import ViteImageOptimizer from 'vite-plugin-image-optimizer'
- Add to plugins array with settings:
  - png: { quality: 80 }
  - jpeg: { quality: 80 }
  - jpg: { quality: 80 }
  - webp: { quality: 80, lossless: false }

Note: The plugin automatically converts JPG/PNG to optimized versions during build. No need for AVIF (Safari support issues).
  </action>
  <verify>
Run `npm run build` - observe image optimization messages in terminal output showing size reductions
Check that dist/assets contains compressed images (significantly smaller than source)
  </verify>
  <done>Build output shows image compression (adobe-max-conference should drop from 2.94MB to under 500KB)</done>
</task>

<task type="auto">
  <name>Task 2: Configure manual chunk splitting for vendor libraries</name>
  <files>vite.config.ts</files>
  <action>
Add rollup manual chunks configuration to vite.config.ts build options:

```typescript
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        'vendor-react': ['react', 'react-dom', 'react-router-dom'],
        'vendor-motion': ['framer-motion', 'lenis'],
        'vendor-ui': [
          '@radix-ui/react-accordion',
          '@radix-ui/react-dialog',
          '@radix-ui/react-slot',
          '@radix-ui/react-toast',
          '@radix-ui/react-switch',
          '@radix-ui/react-label',
          '@radix-ui/react-tabs',
          '@radix-ui/react-tooltip',
        ],
      },
    },
  },
},
```

Only include Radix components that are actually imported in the codebase (accordion, dialog, slot, toast, switch, label, tabs, tooltip are the core ones used by the landing page).
  </action>
  <verify>
Run `npm run build` - output should show separate chunks:
- vendor-react.js (~150KB)
- vendor-motion.js (~120KB)
- vendor-ui.js (~80KB)
- index.js (app code, ~200-300KB)

No chunk should exceed 500KB uncompressed.
  </verify>
  <done>Bundle is split into 4+ chunks with no single chunk exceeding 500KB</done>
</task>

<task type="auto">

Note: rollup-plugin-visualizer is already in devDependencies (checked package.json).

Add visualizer plugin to vite.config.ts (production builds only):
- Import { visualizer } from 'rollup-plugin-visualizer'

  ```typescript

    filename: 'dist/stats.html',
    gzipSize: true,
    brotliSize: true,
  ```
