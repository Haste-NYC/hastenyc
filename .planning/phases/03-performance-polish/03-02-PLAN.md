
3. Update index.html:
- REMOVE the Google Fonts link tags (lines with fonts.googleapis.com and fonts.gstatic.com)
- ADD preload hints before the title tag:
```html
<link rel="preload" href="/fonts/inter-var.woff2" as="font" type="font/woff2" crossorigin>
<link rel="preload" href="/fonts/space-grotesk-var.woff2" as="font" type="font/woff2" crossorigin>
```
- ADD inline @font-face in a style tag (in head, after preloads):
```html
<style>
  @font-face {
    font-family: 'Inter';
    src: url('/fonts/inter-var.woff2') format('woff2');
    font-weight: 100 900;
    font-display: swap;
    font-style: normal;
  }
  @font-face {
    font-family: 'Space Grotesk';
    src: url('/fonts/space-grotesk-var.woff2') format('woff2');
    font-weight: 300 700;
    font-display: swap;
    font-style: normal;
  }
</style>
```

4. Remove any @font-face definitions from src/index.css if they exist (they shouldn't, but check).

  <action>
1. Install web-vitals library:
```bash
npm install web-vitals
```

2. Create src/lib/webVitals.ts:
```typescript
import { onCLS, onINP, onLCP } from 'web-vitals';

export function reportWebVitals() {
  onCLS((metric) => {
    console.log('[CWV] CLS:', metric.value.toFixed(3), metric.value < 0.1 ? '(good)' : '(needs improvement)');
  });

  onINP((metric) => {
    console.log('[CWV] INP:', metric.value, 'ms', metric.value < 200 ? '(good)' : '(needs improvement)');
  });

  onLCP((metric) => {
    console.log('[CWV] LCP:', metric.value.toFixed(0), 'ms', metric.value < 2500 ? '(good)' : '(needs improvement)');
  });
}
```

3. Update src/main.tsx to call reportWebVitals:
- Import: `import { reportWebVitals } from './lib/webVitals';`
- Call after render: `reportWebVitals();`

Note: This logs to console for development verification. In production, you would send to analytics (but that's out of scope for this phase).
  </action>
  <verify>
Run `npm run dev` and open browser console:
- Scroll around the page, interact with elements

- All values should show "(good)" for the thresholds
  </verify>
  <done>Web Vitals logged to console with good/needs-improvement indicators</done>
